How to Use DragonPack's Event System

DragonPack utilizes a custom, dynamic event system for easy game customization with no scripting! This tutorial explains the basics of how to use it, using Dispatch on Keyboard Input and Modify Sprite on Event.

This tutorial follows suggested practices for naming conventions and organization - you may name and organize how you see fit, but make sure it's consistent and everything makes sense!

This is the project that was used to create this tutorial. Feel free to download it to mess around with, but remember that creating things from scratch is important to really learn things!

The Basics

DragonPack has custom event objects (appropriately named: 'EventObject') that you create and name. Some components send events, and other components receive events. The name of your events doesn't matter, as long as the name makes sense to you so you don't get confused. Read the rest of this tutorial to learn how to use them!

Step 1

Create a Unity project and import the DragonPack code package.

Step 2

For organization, create a folder in your Assets directory, and name it 'Events'.

  • Right click in the blank space of the Assets folder in your Project window, and Create -> Folder.

Step 3

Create a folder to organize all of your event objects.

  • Right click in the empty space of your 'Events' folder, and go to Create -> LPK -> EventObject. This is how you make all event objects.

Step 4

Rename the event object to something that makes sense. The name of the event should correspond with the conditions that determine when it gets sent. In this tutorial, we are dispatching an event when the space key is pressed, so the name of the event is 'SpacePressed'.

  • Right click on the event object, and select Rename to name it something that makes sense in the context.

Step 5

Create an empty game object to manage the inputs for the project.

  • Click on GameObject -> Create Empty. Right click on it to rename it GameManager.
  • Add the Dispatch on Keyboard Input component to it.

Step 6

In this project, we want to send an event when the space key is pressed. We need to alter the Dispatch on Keyboard Input component to make this happen.

  • Change the Trigger Key to be 'Space'.
  • Set 'Event' under 'KeyboardInputEvent' to be 'SpacePressed', by clicking on the small circle.
  • Make sure 'Event Sending Mode' is set to ALL. If it's set to OWNER, only components on the same object will be able to receive the event 'SpacePressed'.
  • Check the box next to 'Print Debug Info' to print the event info to the console - if something goes wrong, we can use the console to confirm if the event is actually happening or not.

Step 7

Now, press the play button to run the game, then press the space bar. You should see this message in your console! If you don't, double check your names and component properties.

Step 8

Console messages are cool, but we want something to actually happen in the game when the space bar is pressed! We're going to set things up so that a 2D object has it's sprite change.

  • Click on GameObject -> 2D Object -> Sprite to create a 2D object. Rename it Player.
  • Add the Modify Sprite on Event component to it.

Step 9

We need to modify the Modify Sprite on Event to change the Player's sprite when it receives 'SpacePressed'.

  • Under 'Sprite Properties', change texture mode to 'SET', and 'New Texture' to the sprite you want Player to be when the space bar is pressed. This well set the Player's sprite to the chosen new one!
  • Check the box next to 'Print Debug Info'.
  • Next, we need to tell it what event will trigger these changes. Under Event Receiving Info, change the Event Trigger to SpacePressed.
Now that this is done, our two components are connected! Dispatch on Keyboard Input will send SpacePressed when the space key is pressed, and Modify Sprite on Event will modify our sprite when it receives SpacePressed. An infinite number of components can dispatch and receive the same events, but realistically, you don't want to have a lot of duplicates.

That's It!

Now you know how to use the basics of DragonPack's event system! Each component is slightly different, but the mechanics of event sending and receiving never change.
